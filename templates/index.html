Epigraphic Boolean Search Engine
New to Boolean Search?

In this example EmunIO will find all documents about the Legio IX Hispana (also written Legio nona Hispana or Legio VIIII Hispana), a legion of the Imperial Roman army.

This query combines (AND) 3 sub-queries:

  • "legio OR legionis OR legioni OR legionem OR legione": query the noun "legio" in all possible declension cases.
  • "ix OR viiii or nona or nonae or nonam": query the number 9 both roman numbers and in words (any declension cases).
  • "hispana OR hispanae OR hispanam": query the adjective "hispana" in all possible declension cases.

In this example EmunIO will find all documents about Mithras (also written Mitras or Mitrhas), a deity of the Roman Empire

This query retrieves the word mithras in any form and declension:

  • "mithras OR mithrae OR mithram OR mithra": query the noun "mithras" in all possible declension cases.
  • "mitras OR mitrae OR mitram OR mitra": query the noun "mitras" in all possible declension cases.
  • "mitrhas OR mitrhas OR mitrham OR mitrha: query the noun "mitrhas" in all possible declension cases.

In this example EmunIO will find all documents about Iulia Domna (160–217 AD), a powerfull Roman Empress.

This query combines (AND) 2 sub-queries:

  • "iulia or iuliae or iuliam": query the nomen "iulia" in all possible declension cases.
  • "domna or domnae or domnam": query the cognomen "domna" in all possible declension cases.
New to Boolean Search?

Boolean Search is a type of query that combine keywords and Boolean operators (AND, OR and NOT) to refine results.

AND

Ex: "titius AND gaius" This query will retrieve all documents having both "titius" and "gaius".

OR

Ex: "caesar OR nihil" This query will retrieve all documents containing the word "caesar" or containing the word "nihil" or both.

NOT

Ex: "NOT olet" This query will retrieve all documents except for those who contain the word "olet".

Combine Operators

Boolean search operators can be used in any combination, but there is an order in which the operations are executed:

  • First all NOT operators,
  • Then all OR operators,
  • Finally all AND operators

Ex. "imperator AND tiberius OR imperator AND NOT nero" will be executed in the following order: imperator AND (tiberius OR imperator) AND (NOT nero)

Documentation

EmunIO proposes a boolean search engine and a space-time data visualization for epigraphic databases, features that the author needed but which he did not find. Hence the idea of creating them by himself and proposing them to the community.

The epigraphic database taken into consideration for this project is Epigraphic Database Heidelberg (EDH). Although EmunIO may seem to be a criticism of the Heidelberg platform, it must be considered that EDH is currently the cutting-edge epigraphic database, the only one that has really understood and put into practice the concept of open data.

Boolean Search Engine

The idea of a complex boolean search engine started when the author needed to find all documents talking about Mithras, whose date and place were known, inside EDH to create a space-time data visualization to show the evolution of Mithraism in space and time through epigraphic evidence.

  • Problem 1: multiple boolean queries

    Even if EDH offers a boolean search engine, this is available only with 2 terms ("a and b", "a or b"). To search for Mithras (also written "Mitras" or "Mitrhas") it is necessary combine 3 lemmas: "Mithr" or "Mitr" or "Mitrh". Furthermore, using a query based only on lemmas is very inaccurate (i.e Mithridates is not Mithras). For this reason, considering all the declensions (-as, -ae, -am, -a), there are 12 terms to look for.
    Only one possible solution: query one at a time, download the results and merge them.

  • Problem 2: download data

    As already mentioned, it was necessary to find only the documents whose date and place were known. To do this it was therefore necessary to perform an Advanced Search to apply these two filters. Here a new problem arose: in EDH it is not possible to download the results of an Advanced Search. The project was stuck.

  • Problem 3: SPARQL endpoint

    In EDH a SPARQL Endpoint is available to execute some queries through SPARQL language. Unfortunately the endpoint has a results limit of 100, a lot of data would have been lost.

After encountering all these frustrating problems, the best solution was to create a new search engine to meet the needs of Latin grammar. It was no longer a search for the word "Mithras" but the possibility to create a totally new platform in support of EDH.

  • EmunIO Dataset

    First it was necessary to create a new dataset in csv format to be processed in the boolean search engine.
    The EDH data dump file containing every epigraph metadata ("edh_inscriptions.ttl" inside "edh_linked_data.zip") and the json file containing the geographical coordinates of ancient places ("edhGeographicData.json") were therefore downloaded thanks to EDH OpenData policy.
    The "edh_inscriptions.ttl" file was then loaded on a local endpoint (Blazegraph) and it was processed by the script datamerge.py:

    1. All documents whose date and place were known has been retrieved through a SPARQL query sent to the local endpoint.
    2. For each retrieved document the corresponding place inside the "edhGeographicData.json" file was found, and information about the coordinates were extracted from it.
    3. Finally all the documents were saved in a single csv file: FILE:CSV

  • EmunIO Engine

    The search engine was finally implemented in finder.py:

    1. It processes the csv dataset and converts it into a python list of dictionaries.
    2. It executes the query, without any restriction on its complexity, respecting the order of the Boolean operators (first "NOT, then "OR, finally "AND").
    3. It returns not only the results to be visualized but also a results json file ready to be downloaded by user.

Space-Time Data Visualization

Once the data are ready, it is possible to focus on the visualization.

EDH proposes 2 interesting data interfaces to analyze results:

  • Distribution by chronological data

    • A graph depicting the data frequency distribution over the years.

    PROS: this graph provides really interesting information on an historical point of view in a very clear way.

    CONS: this graph is made using Highchart, a interesting proprietary software, the service is chargeable or it comes free with a Creative Commons Attribution-NonCommercial licence, and this means no open access possibility.

  • Map View Distribution

    • A Map depicting the data distribution over the space.

    PROS: EDH makes great use of the leaflet markercluster library (Open Source library) in managing multiple documents in the same place.

    CONS: On an historical study point of view having a map that shows all the documents without making a distinction in the time period greatly reduces the possibilities for analysis.

EmunIO Data Visualization

Starting from EDH platform the idea was really simple: merge the two interfaces into one. A timeline slider to select date which not only shows the frequency of documents on that date but also displays them on a map. A similar approach can be seen in Peripleo but in this example markers inside maps aren't clustered, so the interface has problem to handle overlapping geographical information. On the other hand EmunIO Data Visualization interface on has the following unique features:

  • Totally Open Source code (including external libraries)
  • Markers clustered inside map
  • Document's data available directly inside marker's pop-up
  • Multiple navigation possibilities:
    • moving slider
    • clicking arrows (forward backwards)
    • typing the date in the box
    • clicking directly on the chart
  • Auxiliary analysis tools:
    • display total result count
    • percentage of epigraphs by selected year
    • line of annual average of epigraphs diplayed on the chart

Future Developments

EmunIO is not complete.
By relating space and time in an interface, the importance of the concept of historical evolution in the context of epigraphy was highlighted. The next step will be to bring out the evolution also in the political geography of the map. The expansion and decline of Rome represent the context of the dissemination of epigraphic documents. In example, there are documents in Dacia before the official annexation of the Roman Empire, and this issue has a value to emphsize. To do this, however, an important research for sources and atlases to be digitized is required, always respecting Open Definition principles.
